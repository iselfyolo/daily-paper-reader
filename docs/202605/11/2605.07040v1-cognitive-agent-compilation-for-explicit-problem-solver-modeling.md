## 摘要
大语言模型 (LLMs) 被广泛用于辅导、反馈生成和内容创作，但其广泛的预训练使其难以约束，且难以作为可控学习者的替代品。教育系统通常需要可检查且可编辑的知识状态：教育者希望了解系统假设学习者掌握了什么，而当系统能够根据显式技能、误解和策略来解释其行为时，学习者也会从中受益。受认知架构启发，我们提出了认知智能体编译 (CAC)，这是一个利用强大的教师 LLM 将问题求解知识编译到显式目标智能体中的框架。CAC 将 (i) 知识表示、(ii) 问题求解策略以及 (iii) 验证和更新规则分离，旨在使教育场景中的有界问题求解更具可检查性和可编辑性。我们展示了一个使用小语言模型 (SLMs) 实现的早期概念验证，揭示了关键的设计权衡，特别是显式控制与可扩展泛化之间的权衡，并将 CAC 定位为迈向教育应用中有界知识人工智能的初步尝试。

## Abstract
Large language models (LLMs) are widely used for tutoring, feedback generation, and content creation, but their broad pretraining makes them hard to constrain and poor substitutes for controllable learners. Educational systems often require inspectable and editable knowledge states: educators want to know what a system assumes the learner knows, and learners benefit when the system can justify actions in terms of explicit skills, misconceptions, and strategies. Inspired by cognitive architectures, we propose Cognitive Agent Compilation (CAC), a framework that uses a strong teacher LLM to compile problem-solving knowledge into an explicit target agent. CAC separates (i) knowledge representation, (ii) problem-solving policy, and (iii) verification and update rules, with the goal of making bounded problem solving more inspectable and editable in educational settings. We present an early proof of concept implemented with Small Language Models that surfaces key design trade-offs, particularly between explicit control and scalable generalization, and positions CAC as an initial step toward bounded-knowledge AI for educational applications.

*   **研究动机**：虽然 LLM 在教育领域应用广泛，但它们存在“知识抑制”困难，即很难模拟一个“不知道某些知识”的学习者状态。此外，LLM 的推理过程是黑盒化的，而教育系统需要可检查、可编辑的知识状态（如显式的技能、误解和策略）。
*   **核心目标**：受认知架构（如 ACT-R, SOAR）启发，作者希望利用强大的“教师 LLM”将解题知识“编译”到一个受限的、显式的“目标智能体”中，使其行为完全基于可解释的知识库，从而支持更精准的学习者建模。

### 2. 方法论
CAC 框架的核心思想是将知识表示、解题策略和更新规则分离，通过一个**失败驱动的循环**进行编译：
*   **核心组件**：
    *   **教师 LLM**：作为诊断和编译引擎，负责分析失败原因并生成显式知识。
    *   **认知智能体（目标智能体）**：基于小语言模型（SLM）构建，严格依赖显式知识库（KB）进行推理。
    *   **显式知识库（KB）**：存储人类可读的知识项（KCs）、适用条件和更新规则。
*   **算法流程（CAC 循环）**：
    1.  **尝试解题**：智能体仅依靠当前 KB 尝试解决问题，记录详细的推理轨迹。
    2.  **失败分析**：教师 LLM 检查轨迹，识别缺失的知识组件或触发误解的因素。
    3.  **更新知识库**：教师 LLM 生成新的显式知识条目（如规则、程序草图）并注入 KB。
    4.  **重复验证**：智能体使用更新后的 KB 重新尝试，直到成功且推理过程符合约束。
*   **技术细节**：采用 **QKV（Query-Key-Value）机制**进行知识检索。智能体的目标和工作记忆（WM）作为 Query，KB 中的适用条件作为 Key，知识内容作为 Value。

### 3. 实验设计
*   **数据集/场景**：使用了 **OLI_Biology 数据集**（来自 CMU DataShop），该数据集包含生物学多选题、学生交互日志和课程内容，具有较强的陈述性记忆依赖性。
*   **实验设置**：
    *   **目标智能体**：Qwen2.5-2B（负责执行受限指令）。
    *   **检索模型**：embedding-gemma (300M)。

*   **对比与基准**：本研究属于概念验证（Proof of Concept），主要验证框架的可行性，而非与特定 SOTA 模型进行性能对比。实验观察了智能体在 27 个生物学问题上的编译效率和知识积累过程。

### 4. 资源与算力
*   **算力使用**：论文提到使用了**本地运行**的 27B 参数模型（gemma-3-27b-it），并采用了 **4-bit 量化**技术以降低显存需求。
*   **具体配置**：未明确说明具体的 GPU 型号、数量或总训练时长，但强调了使用小语言模型（SLM）是为了推理效率和微调的灵活性。

### 5. 实验数量与充分性
*   **实验规模**：对 27 个生物学问题进行了迭代编译，共生成了 188 条陈述性记忆条目，平均每个问题经历 3.1 次编译迭代。
*   **充分性评价**：作为早期概念验证，实验规模较小。作者坦诚实验在第 28 个问题时因“扇出效应”（Fan Effect）导致检索失效而终止。目前尚未在大规模数据集或真实学生数据上进行交叉验证，因此实验的普适性有待进一步加强。

### 6. 主要结论与发现
*   **解耦可行性**：证明了可以将目标管理、工作记忆更新和知识检索在受限管道中部分分离。
*   **SLM 的潜力**：在严格约束的动作空间下，2B 规模的 SLM 足以担任认知智能体的执行器。
*   **检索瓶颈**：随着知识库增大，基于相似性的检索会出现精度下降（扇出效应），导致智能体陷入死循环。
*   **教师模型效能**：即使是量化后的中等规模本地模型（27B），也能有效执行复杂的诊断和知识注入任务。

### 7. 优点
*   **高可解释性**：将 LLM 的隐式推理转化为显式的知识条目，使 AI 的“知识状态”变得可观察、可编辑。
*   **理论结合紧密**：成功将经典的认知架构理论（如 ACT-R 的生产规则和陈述性记忆）与现代 LLM 的生成能力相结合。
*   **资源友好**：展示了使用中小型本地模型构建复杂教学系统的可能性，降低了对昂贵 API 的依赖。

### 8. 不足与局限
*   **先验知识干扰**：SLM 内部仍残留预训练的先验知识，难以完全保证其解题仅依赖于外部注入的 KB。
*   **可扩展性问题**：简单的向量检索难以处理大规模知识库，容易出现检索错误或过度泛化。
*   **协作作弊风险**：教师 LLM 有时会直接给出答案而非生成教学知识，需要更严格的程序化过滤。
*   **缺乏人类验证**：尚未证明该模型生成的知识组件（KCs）能准确预测真实人类学生的表现。

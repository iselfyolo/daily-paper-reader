## 摘要
随着模型家族、训练方案和计算预算日益标准化，机器学习系统的进一步提升越来越依赖于数据。然而，数据工程在很大程度上仍是手动且随机的：从业者反复搜索外部数据集，将其适配到现有流水线，通过下游训练验证候选数据，并从之前的尝试中总结经验。我们研究了任务条件下的自主数据工程，其中自主智能体通过仅优化数据侧（包括外部数据发现、数据选择与组合、清洗与转换）来改进固定的学习算法。其目标是在保持学习算法不变的情况下，获得更强大的下游解决方案。为了应对自主数据工程中固有的开放式搜索空间、依赖分支的细化以及延迟验证等挑战，我们提出了 DataMaster，这是一个集成了树状结构搜索、共享候选数据和累积记忆的数据智能体框架。DataMaster 由三个核心组件组成：组织备选数据工程分支的 DataTree、存储已发现外部数据源以供重用的共享 Data Pool（数据池），以及记录节点结果、中间产物和可重用发现的 Global Memory（全局记忆）。这些组件共同协作，使智能体能够发现候选数据，构建可执行的训练输入，通过下游反馈对其进行评估，并在不同分支间传递有用证据。我们在两类基准测试 MLE-Bench Lite 和 PostTrainBench 上评估了 DataMaster。在 MLE-Bench Lite 上，它比初始分数提高了 32.27% 的奖牌率；在 PostTrainBench 上，它在 GPQA 上的表现超过了指令模型（31.02% 对 30.35%）。

## Abstract
As model families, training recipes, and compute budgets become increasingly standardized, further gains in machine learning systems depend increasingly on data. Yet data engineering remains largely manual and ad hoc: practitioners repeatedly search for external datasets, adapt them to existing pipelines, validate candidate data through downstream training, and carry forward lessons from prior attempts. We study task-conditioned autonomous data engineering, where an autonomous agent improves a fixed learning algorithm by optimizing only the data side, including external data discovery, data selection and composition, cleaning and transformation. The goal is to obtain a stronger downstream solution while leaving the learning algorithm unchanged. To address the open-ended search space, branch-dependent refinement, and delayed validation inherent in autonomous data engineering, we propose DataMaster, a data-agent framework that integrates tree-structured search, shared candidate data, and cumulative memory. DataMaster consists of three key components: a DataTree that organizes alternative data-engineering branches, a shared Data Pool that stores discovered external data sources for reuse, and a Global Memory that records node outcomes, artifacts, and reusable findings. Together, these components allow the agent to discover candidate data, construct executable training inputs, evaluate them through downstream feedback, and carry useful evidence across branches. We evaluate DataMaster on two types of benchmarks, MLE-Bench Lite and PostTrainBench. On MLE-Bench Lite, it improves medal rate by 32.27% over the initial score; on PostTrainBench, it surpasses the instruct model on GPQA (31.02% vs 30.35%).

### DataMaster：迈向机器学习的自主数据工程——论文深度总结

#### 1. 论文的核心问题与整体含义
随着预训练模型、训练算法和算力资源的日益标准化，机器学习（ML）系统的性能提升瓶颈逐渐转移到**数据侧**。然而，目前的数据工程（包括外部数据搜索、清洗、转换、配比等）仍高度依赖人工经验，且过程零散、难以复用。
论文提出了**任务条件下的自主数据工程（Task-conditioned Autonomous Data Engineering）**这一新问题：即在保持学习算法和模型代码不变的前提下，如何让自主智能体（Agent）通过优化数据状态来最大化下游任务性能。

#### 2. 论文提出的方法论
论文引入了 **DataMaster** 框架，其核心思想是将数据工程建模为一个受下游训练反馈驱动的**树状搜索过程**。
*   **核心组件：**
    *   **DataTree（数据树）：** 组织不同的数据工程分支。包含两种节点：
        *   **红节点（Red Nodes）：** 负责“探索”，通过网络搜索、API 调用等手段发现外部数据集，并将其元数据存入共享池。
        *   **黑节点（Black Nodes）：** 负责“利用”，从数据池中选择数据，编写清洗、转换和特征工程代码，构建可执行的 DataLoader，并触发下游训练以获取真实反馈。
    *   **Data Pool（共享数据池）：** 存储所有已发现的外部数据源，供不同分支的黑节点复用。
    *   **Global Memory（全局记忆）：** 记录所有节点的尝试结果、失败原因、成功的处理逻辑和性能反馈，实现跨分支的知识积累。
*   **调度策略：** 采用基于 **UCB1（置信区间上界）** 算法的调度器，在有限的计算预算下平衡对新数据源的探索和对已有成功路径的深化。

#### 3. 实验设计
论文在两个具有代表性的基准测试上验证了 DataMaster 的有效性：
*   **MLE-Bench Lite：** 包含 22 个 Kaggle 竞赛任务（涵盖表格、视觉、NLP、时间序列等）。Agent 需要在给定初始代码的基础上，仅通过数据操作提升排名。
*   **PostTrainBench：** 评估 LLM 的后训练能力。Agent 不获赠任何训练数据，必须从零开始发现并构建数据集来提升基础模型（如 Qwen3-1.7B-Base）在 AIME、GPQA、GSM8K 等 7 个维度的表现。
*   **对比方法：** 包括通用编程 Agent（Claude Code, Codex）、专门的数据搜索 Agent（DatasetResearch）、端到端 ML Agent（ML-Master 2.0）以及数据中心化训练框架（DataFlex）。

#### 4. 资源与算力
*   **硬件环境：** 使用 8 路 NVIDIA **H20 GPU** 机器。在 MLE-Bench Lite 实验中，每 4 个任务共享 1 个 GPU 进行并行实验。
*   **时间预算：** 每个任务设定了 **12 小时** 的时间限制。
*   **模型底座：** 主要实验基于 **GLM-5**，消融实验对比了 Claude-4.6-sonnet、Qwen3.5-Plus 和 GPT-5.4。
*   **成本分析：** 论文详细记录了 API 调用成本，黑节点由于涉及实际训练和验证，其 API 成本（约 $0.062/节点）和耗时（约 50.6 分钟/节点）显著高于红节点。

#### 5. 实验数量与充分性
*   **实验规模：** 涵盖了 29 个不同的 ML 任务（22 个 Kaggle + 7 个 LLM 能力项）。在 MLE-Bench Lite 上共执行了 981 个节点尝试。
*   **消融实验：** 针对红/黑节点的功能、全局记忆的有无、以及初始代码质量（全代码 vs 仅算法框架）进行了多组对比实验。
*   **客观性与公平性：** 实验设置了严格的**数据泄露检查协议**（包括 URL 过滤、哈希去重、3-5 gram 重叠度分析），确保 Agent 提升的性能并非来自对测试集的偷窥。对比基准均在相同的算力和时间约束下运行。

#### 6. 论文的主要结论与发现
*   **性能显著提升：** 在 MLE-Bench Lite 上，DataMaster 将奖牌率提升了 **32.27%**；在 PostTrainBench 上，平均分提升了 **19.79** 点。
*   **超越人类专家：** 在 GPQA（研究生水平科学问题）任务中，DataMaster 构建的数据集训练出的模型性能（31.02%）超过了官方发布的指令微调模型（30.35%）。
*   **记忆的重要性：** 消融实验显示，如果没有全局记忆，Agent 会陷入重复失败，奖牌率提升几乎归零。
*   **测试时缩放（Test-time Scaling）：** 随着搜索时间和训练预算的增加，模型性能呈现阶梯式上升，证明了该框架能有效转化算力为数据质量。

#### 7. 优点（亮点）
*   **闭环优化：** 不同于以往只做数据清洗或只做搜索的工作，DataMaster 将“搜索-处理-训练-反馈”整合进一个闭环。
*   **结构化搜索：** DataTree + UCB 调度的设计比简单的线性迭代更科学，能有效处理数据工程中分支依赖的问题。
*   **通用性强：** 框架同时适用于传统 ML 竞赛和前沿的 LLM 后训练场景。

#### 8. 不足与局限
*   **对底座模型依赖高：** 实验发现 GPT-5.4 由于工具调用频率低，在该框架下表现较差，说明 DataMaster 的效能高度依赖于底座 LLM 的 Agent 能力。
*   **计算成本：** 虽然比人工快，但频繁的下游训练验证（黑节点）依然消耗大量计算资源。
*   **搜索空间的潜在风险：** 尽管有泄露检查，但在开放互联网搜索数据时，如何完全排除“训练集包含测试集变体”的隐性泄露仍是一个长期挑战。
*   **应用限制：** 目前主要针对有明确评估指标（Metric）的任务，对于缺乏自动评估手段的开放式生成任务，其反馈回路可能失效。

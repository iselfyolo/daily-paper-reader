## 摘要
长程对话智能体依赖于具有日益复杂检索机制的记忆系统。然而，检索到的片段通常以非结构化文本的形式输入语言模型，缺乏复杂推理所必需的关系、时间和主题结构。为了弥补这一推理差距，我们引入了 GRAVITY（通过注入拓扑记忆实现的生成时关系锚定），这是一种即插即用的结构化记忆模块。GRAVITY 从原始对话话语中提取三种互补的知识表示：基于关系图的实体概况、链接到因果轨迹的时间事件元组，以及跨会话的主题摘要。在生成时，它将这些表示作为结构化锚定上下文注入宿主系统的提示词中。该方法有效地将分散的证据合成为连贯且与查询相关的上下文，而无需对宿主模型进行任何架构修改。在 LongMemEval 和 LoCoMo 基准测试上对五种不同记忆系统进行的广泛评估证明了我们方法的有效性。平均而言，GRAVITY 将 LLM 评判准确率提高了 7.5%--10.1%。增益与基线强度呈负相关：最弱的宿主提升了 12.2%，而最强的宿主仍获得了 3.8%--5.7% 的提升。这些发现确立了结构化上下文锚定作为一种广泛有效、架构无关的长程对话记忆增强范式。

## Abstract
Long-horizon conversational agents rely on memory systems with increasingly sophisticated retrieval mechanisms. However, retrieved fragments are typically fed to the language model as unstructured text, lacking the relational, temporal, and thematic structures essential for complex reasoning. To bridge this reasoning gap, we introduce GRAVITY (\textbf{G}eneration-time \textbf{R}elational \textbf{A}nchoring \textbf{V}ia \textbf{I}njected \textbf{T}opological Memor\textbf{Y}), a plug-and-play structured memory module. GRAVITY extracts three complementary knowledge representations from raw conversational utterances: entity profiles grounded in relational graphs, temporal event tuples linked into causal traces, and cross-session topic summaries. At generation time, it injects these representations into the host system's prompt as structured anchoring contexts. This approach effectively synthesizes scattered evidence into a coherent, query-relevant context without requiring any architectural modifications to the host model. Extensive evaluations across five diverse memory systems on the LongMemEval and LoCoMo benchmarks demonstrate the efficacy of our approach. On average, GRAVITY improves LLM-judge accuracy by 7.5--10.1%. Gains are inversely correlated with baseline strength: the weakest host improves by 12.2% while the strongest still gains 3.8--5.7%. These findings establish structured context anchoring as a broadly effective, architecture-agnostic augmentation paradigm for long-horizon conversational memory.

#### 1. 核心问题与整体含义
*   **研究背景**：长程对话智能体（Long-horizon agents）需要跨越数百个会话维持连贯性。目前的记忆系统主要依赖检索增强生成（RAG），将检索到的原始文本片段喂给大模型（LLM）。
*   **核心痛点**：作者发现检索与生成之间存在“推理鸿沟”。即使检索到了正确的信息，由于这些片段是零散、非结构化的，LLM 难以自动重建其中的**实体关系（Relational）**、**时间顺序（Temporal）**和**主题逻辑（Thematic）**，导致在多跳推理和时间推理任务上表现不佳。
*   **研究目的**：提出一种无需修改现有记忆系统架构、即插即用的模块，通过注入结构化知识来增强 LLM 的推理能力。

#### 2. 方法论
GRAVITY 的核心思想是**生成时结构化锚定（Generation-time Structured Anchoring）**。
*   **核心组件（三维锚点）**：
    1.  **实体锚点 (Entity Anchors)**：构建动态实体画像，包含属性、类型、与其他实体的关系以及状态演变的时间线。
    2.  **事件锚点 (Event Anchors)**：提取 **4W1O**（Who, What, When, Where, Outcome）元组，并将相关事件链接为“时间轨迹”（Temporal Traces）。
    3.  **主题锚点 (Topic Anchors)**：进行跨会话的主题聚合，生成宏观叙事摘要，捕捉片段记忆无法表达的长程弧线。
*   **算法流程**：
    *   **构建阶段（离线）**：从原始对话中批量提取上述三类锚点，独立于宿主系统的索引。
    *   **推理阶段（在线）**：
        1.  **检索与重排**：基于查询检索相关锚点，并使用嵌入向量进行重排，同时引入“时间保护机制”确保时间相关事件不被过滤。
        2.  **查询扩展**：利用锚点内容生成扩展查询，帮助宿主系统检索更广泛的原始记忆。
        3.  **上下文注入**：将 Top-K 结构化锚点以特定格式注入 Prompt，作为 LLM 生成答案的“脚手架”。

#### 3. 实验设计
*   **数据集**：
    *   **LoCoMo**：包含 1540 个 QA 对，侧重长程社交对话中的多跳、事实召回和时间推理。
    *   **LongMemEval**：包含 500 个问题，涵盖 7 类任务（如知识更新、不确定性拒绝等），对话长度最高达 115 个会话。
*   **对比方法（宿主系统）**：将 GRAVITY 插入 5 种不同范式的记忆系统：

*   **评测基准**：使用 GPT-4o-mini 作为 LLM-judge，评估回答的准确率。

    *   推理阶段：平均增加 0.31s - 0.92s 的延迟，额外消耗约 2K tokens。

#### 5. 实验数量与充分性
*   **实验规模**：涵盖了 5 种主流架构和 2 个大型基准测试，实验结果具有普适性。
*   **消融实验**：详细对比了单一模块（E/V/T）的效果、查询扩展的作用、以及重排机制的重要性。

*   **Oracle 实验**：在“完美检索”的假设下验证了结构化信息在抗噪声和辅助推理中的独特价值。

#### 6. 主要结论与发现
*   **显著提升**：GRAVITY 使各系统的平均准确率提升了 **7.5% - 10.1%**。
*   **负相关增益**：基线越弱的系统受益越大（最高提升 12.2%），而最强的基线（LightMem）仍能获得约 5% 的稳健提升。

#### 7. 优点
*   **架构无关性**：真正的即插即用，无需重新训练模型或修改底层数据库索引。
*   **多维建模**：系统性地解决了关系、时间和主题三个维度的结构缺失。
*   **灵活性与效率**：支持并行提取和开源小模型替代，提供了多种成本优化方案。
*   **理论支撑**：通过 Oracle 实验和数学推导证明了结构化信息在处理检索噪声时的“引力锚定”作用。

#### 8. 不足与局限
*   **过度总结风险**：在提取过程中可能丢失极细粒度的细节，导致模型回答过于笼统。
*   **时间解析偏差**：对于复杂的相对时间引用，提取模块有时会分配错误的绝对时间槽。
*   **依赖提取质量**：如果构建阶段的 LLM 产生幻觉或错误关联，会通过锚点误导生成阶段。
*   **推理延迟**：虽然增量较小，但在实时性要求极高的场景下，额外的锚点检索和 Prompt 长度仍是负担。

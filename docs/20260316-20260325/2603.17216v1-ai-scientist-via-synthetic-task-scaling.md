## 摘要
随着 AI 智能体的出现，自动化科学发现已成为一个可行的目标。许多近期工作构建了能够进行机器学习研究的智能体系统，但并未提供训练此类智能体的原则性方法，且当前的 LLM 往往会生成看似合理但无效的想法。为了在训练能够“从实践中学习”的智能体方面取得进展，我们提供了一种针对机器学习智能体的新型合成环境生成流水线。该流水线能够自动合成与 SWE-agent 框架兼容的机器学习挑战，涵盖了主题采样、数据集提议和代码生成。生成的合成任务具有以下特点：1) 基于真实的机器学习数据集，因为提议的数据集已通过 Huggingface API 验证；2) 通过自调试循环验证，确保了更高的质量。为了验证合成任务的有效性，我们在机器学习任务基准测试 MLGym 上进行了实验。我们从合成任务中采样教师模型（GPT-5）的轨迹，并利用这些轨迹训练学生模型（Qwen3-4B 和 Qwen3-8B）。实验结果表明，使用合成任务训练的学生模型在 MLGym 上的性能显著提升，Qwen3-4B 的 AUP 指标提高了 9%，Qwen3-8B 提高了 12%。

## Abstract
With the advent of AI agents, automatic scientific discovery has become a tenable goal. Many recent works scaffold agentic systems that can perform machine learning research, but don't offer a principled way to train such agents -- and current LLMs often generate plausible-looking but ineffective ideas. To make progress on training agents that can learn from doing, we provide a novel synthetic environment generation pipeline targeting machine learning agents. Our pipeline automatically synthesizes machine learning challenges compatible with the SWE-agent framework, covering topic sampling, dataset proposal, and code generation. The resulting synthetic tasks are 1) grounded in real machine learning datasets, because the proposed datasets are verified against the Huggingface API and are 2) verified for higher quality with a self-debugging loop. To validate the effectiveness of our synthetic tasks, we tackle MLGym, a benchmark for machine learning tasks. From the synthetic tasks, we sample trajectories from a teacher model (GPT-5), then use the trajectories to train a student model (Qwen3-4B and Qwen3-8B). The student models trained with our synthetic tasks achieve improved performance on MLGym, raising the AUP metric by 9% for Qwen3-4B and 12% for Qwen3-8B.

*   **背景**：虽然现有的 LLM 具备丰富的 ML 理论知识，但在执行多步、目标导向的科研任务时，往往只能生成“看起来合理”但实际无效的想法。

*   **核心目标**：建立一个可扩展的流水线，通过自动合成大量机器学习任务和执行轨迹，让智能体在“实践中学习”，从而提升其解决复杂科研问题的能力。

### 2. 核心方法论
研究者提出了一种**合成任务扩展（Synthetic Task Scaling）**流水线，分为三个阶段：
*   **阶段 1：环境合成**：
    1.  **主题采样**：从模型中采样 $n$ 个不同的机器学习主题。
    2.  **任务与数据集提议**：针对每个主题，由教师模型提议任务描述和 HuggingFace 数据集。通过 HuggingFace API 验证数据集真实性，并获取数据示例。
    3.  **代码生成**：生成与 SWE-agent 框架兼容的配置文件、基准实现代码和评估脚本。
*   **阶段 2：环境验证与自调试**：
    *   使用 GPT-5 智能体对生成的任务进行“试运行”。如果运行出错，系统会将错误反馈给模型进行**自调试（Self-debugging）**，最多迭代 $k$ 次。无法通过验证的任务将被舍弃，确保了训练数据的质量。
*   **阶段 3：轨迹生成与过滤**：
    *   在高性能计算（HPC）集群上并行运行验证通过的任务，收集教师模型（GPT-5）解决任务的完整轨迹。
    *   **过滤机制**：仅保留至少有一次成功提交且长度适中（<48K tokens）的轨迹。

### 3. 实验设计
*   **基准测试（Benchmark）**：使用 **MLGym**，包含 13 个涵盖计算机视觉、强化学习、语言建模等领域的机器学习挑战。
*   **对比方法**：
    *   **教师模型**：GPT-5。
    *   **基准模型**：GPT-4o、原始 Qwen3-4B 和 Qwen3-8B。
    *   **实验组**：经过合成轨迹微调（SFT）后的 SFT-Qwen3-4B 和 SFT-Qwen3-8B。
*   **评估指标**：AUP（Area Under Performance curve，性能曲线下面积），综合衡量智能体在多次尝试中的优化能力。

### 6. 主要结论与发现
*   **性能显著提升**：经过合成任务训练后，Qwen3-4B 的 AUP 提升了 **9%**，Qwen3-8B 提升了 **12%**。
*   **小模型逆袭**：微调后的 8B 模型在多个任务上的表现接近甚至超过了更强大的基准模型。
*   **迭代能力增强**：实验证明，通过学习包含调试和失败过程的完整轨迹，智能体能更有效地进行迭代优化，而不仅仅是编写初始代码。

### 7. 优点与亮点
*   **完全自动化**：整个流水线无需人工干预，从主题采样到任务验证均由 AI 完成，具有极高的可扩展性。
*   **真实性保障**：通过 HuggingFace API 挂钩真实数据集，避免了纯合成数据可能带来的“幻觉”任务。
*   **闭环质量控制**：引入自调试循环（Self-debugging loop），显著降低了合成环境中的代码崩溃率。
*   **兼容性强**：适配 SWE-agent 框架，意味着该方法可以轻易迁移到其他软件工程或科研自动化任务中。

### 8. 不足与局限
*   **基准局限性**：仅在 MLGym 上验证，可能存在“格式对齐”而非“能力提升”的风险（即模型学会了 MLGym 的特定提交格式）。
*   **教师模型依赖**：学生模型的上限受限于教师模型（GPT-5）的水平，无法超越教师未曾展示的能力。
*   **缺乏创新性优化**：目前的 SFT 范式主要是在模仿，尚未引入强化学习（RL）来显式鼓励智能体发现“全新的科研想法”。
*   **复杂任务覆盖不足**：在处理如 MS-COCO 等需要复杂初始代码库的任务时，性能提升不明显，说明合成流水线在生成复杂架构方面仍有挑战。

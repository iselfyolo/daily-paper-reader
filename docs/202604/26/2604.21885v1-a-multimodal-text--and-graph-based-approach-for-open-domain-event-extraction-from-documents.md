## 摘要
事件抽取对于事件理解和分析至关重要。它支持诸如文档摘要和应急场景决策等任务。然而，现有的事件抽取方法存在局限性：(1) 封闭域算法受限于预定义的事件类型，因此很难泛化到未见过的类型；(2) 开放域事件抽取算法虽然能够处理不受约束的事件类型，但尽管大语言模型（LLMs）具有先进的能力，这些算法在很大程度上忽视了它们的潜力。此外，它们没有显式地建模文档级的上下文、结构和语义推理，这对于有效的事件抽取至关重要，但由于“迷失在中间”（lost-in-the-middle）现象和注意力稀释，这对 LLMs 来说仍然具有挑战性。为了解决这些局限性，我们提出了多模态开放域事件抽取（MODEE），这是一种新型的开放域事件抽取方法，它将基于图的学习与来自 LLMs 的基于文本的表示相结合，以建模文档级推理。在大型数据集上的实证评估表明，MODEE 优于最先进的开放域事件抽取方法，并且可以泛化到封闭域事件抽取，其表现优于现有算法。

## Abstract
Event extraction is essential for event understanding and analysis. It supports tasks such as document summarization and decision-making in emergency scenarios. However, existing event extraction approaches have limitations: (1) closed-domain algorithms are restricted to predefined event types and thus rarely generalize to unseen types and (2) open-domain event extraction algorithms, capable of handling unconstrained event types, have largely overlooked the potential of large language models (LLMs) despite their advanced abilities. Additionally, they do not explicitly model document-level contextual, structural, and semantic reasoning, which are crucial for effective event extraction but remain challenging for LLMs due to lost-in-the-middle phenomenon and attention dilution. To address these limitations, we propose multimodal open-domain event extraction, MODEE , a novel approach for open-domain event extraction that combines graph-based learning with text-based representation from LLMs to model document-level reasoning. Empirical evaluations on large datasets demonstrate that MODEE outperforms state-of-the-art open-domain event extraction approaches and can be generalized to closed-domain event extraction, where it outperforms existing algorithms.

### 2. 方法论：MODEE 框架
MODEE 由四个核心模块组成，采用多模态（文本+图）融合策略：
*   **文本编码器（Text Encoder）**：使用 T5 的编码器部分，将文档 token 序列转化为上下文相关的文本嵌入（$H_{text}$）。
*   **图编码器（Graph Encoder）**：
    *   **构图**：将文档中所有 token 视为节点，构建一个**全连接图（Complete Graph）**，以捕捉长程依赖。
    *   **特征提取**：利用两层 **GraphSAGE**（配合 LSTM 聚合）生成节点级的图嵌入（$H_{graph}$），建模文档的结构化语义。
*   **注意力门控多模态融合（Attention-Based Gated Fusion）**：
    *   通过加性注意力机制计算 token 级的门控向量（$\alpha$）。
    *   将文本嵌入与门控向量相乘，突出显示与事件高度相关的 token，生成集成多模态嵌入（$H_{integrated}$）。
*   **文本解码器（Text Decoder）**：基于 T5 解码器，以自回归方式生成事件的 **5Ws**（Where, When, What, Who, Why）。
*   **训练策略**：采用联合优化。文本端使用交叉熵损失；图端引入**对比学习损失（Contrastive Loss）**，使属于同一 5W 类别的 token 在嵌入空间中更接近。

### 5. 实验数量与充分性
*   **实验规模**：进行了主性能对比、各 5W 类别细分分析、消融实验以及跨领域（闭域）泛化实验。
*   **充分性评价**：实验设计较为全面。不仅对比了同规模的微调模型，还对比了参数量大得多的 70B 模型。消融实验分别验证了对比学习、门控融合机制和全连接图结构的必要性。

### 6. 主要结论与发现
*   **MODEE 性能卓越**：MODEE-Base 在所有指标上均优于微调的 T5-Large 和其他 LLM，证明了图结构信息的补充作用。
*   **攻克“Why”难题**：在最具挑战性、语义最复杂的“Why”（原因）类别上，MODEE 提升显著，这归功于其对文档级上下文的深度建模。
*   **泛化能力强**：在闭域数据集 DocEE 上，MODEE 同样刷新了 SOTA 记录，证明其架构具有普适性。
*   **提示词工程局限**：实验显示，单纯依靠 Prompting（即使是 70B 模型）在复杂的事件抽取任务上表现远不如经过专门设计的微调模型。

### 7. 优点与亮点
*   **单源多模态**：创新性地从纯文本中衍生出“图”这一模态，无需外部图像或音频，降低了数据获取成本。
*   **结构化推理**：通过全连接图和 GraphSAGE 显式解决了 LLM 在长文本中注意力稀释的问题。
*   **端到端生成**：将图特征无缝集成到生成式框架中，直接输出结构化 5Ws，流程简洁高效。

### 8. 不足与局限
*   **任务假设**：目前假设“一文一事件”（One-event-per-document），对于包含多个显著事件的复杂文档处理能力有限。
*   **计算开销**：构建和处理文档级的全连接 token 图在文档极长时会带来显著的计算负担，扩展性受限。
*   **模型底座**：主要基于 T5（Encoder-Decoder 架构），尚未在目前主流的纯 Decoder 架构（如 GPT-4 级别）上验证其图增强的效果。
*   **领域覆盖**：实验主要集中在新闻领域，在社交媒体或专业技术文档上的表现尚待验证。

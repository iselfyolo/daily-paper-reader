## 摘要
小语言模型因其低成本、推理快以及易于专业化等优点，在生产部署中极具吸引力。然而，将它们适配到特定任务仍然是一个具有挑战性的工程闭环，其难点不在于训练本身，而在于周边的决策：数据精选、故障诊断、规避回归以及迭代控制。我们提出了 Pioneer Agent，一个能够自动化这一生命周期的闭环系统。在冷启动模式下，仅需提供自然语言任务描述，该智能体即可获取数据、构建评估集，并通过联合优化数据、超参数和学习策略来迭代训练模型。在生产模式下，给定一个带有标注故障的已部署模型，它会诊断错误模式，构建有针对性的训练数据，并在显式回归约束下进行重新训练。为了评估这一设置，我们引入了 AdaptFT-Bench，这是一个包含噪声逐渐增加的合成推理日志基准，旨在测试完整的适配闭环：诊断、课程合成、重训和验证。在涵盖推理、数学、代码生成、摘要和分类的八个冷启动基准测试中，Pioneer Agent 相比基础模型提升了 1.6-83.8 分。在 AdaptFT-Bench 上，它在所有七个场景中都提升或保持了性能，而朴素的重训则导致性能下降高达 43 分。在基于公共基准任务构建的两个生产级部署中，它将意图分类准确率从 84.9% 提高到 99.3%，将实体 F1 分数从 0.345 提高到 0.810。除了性能提升外，该智能体通常能纯粹根据下游反馈发现有效的训练策略，包括思维链监督、特定任务优化以及关注质量的数据精选。

## Abstract
Small language models are attractive for production deployment due to their low cost, fast inference, and ease of specialization. However, adapting them to a specific task remains a challenging engineering loop, driven not by training itself but by surrounding decisions: data curation, failure diagnosis, regression avoidance, and iteration control. We present Pioneer Agent, a closed-loop system that automates this lifecycle. In cold-start mode, given only a natural-language task description, the agent acquires data, constructs evaluation sets, and iteratively trains models by jointly optimizing data, hyperparameters, and learning strategy. In production mode, given a deployed model with labeled failures, it diagnoses error patterns, constructs targeted training data, and retrains under explicit regression constraints. To evaluate this setting, we introduce AdaptFT-Bench, a benchmark of synthetic inference logs with progressively increasing noise, designed to test the full adaptation loop: diagnosis, curriculum synthesis, retraining, and verification. Across eight cold-start benchmarks spanning reasoning, math, code generation, summarization, and classification, Pioneer Agent improves over base models by 1.6-83.8 points. On AdaptFT-Bench, it improves or preserves performance in all seven scenarios, while naive retraining degrades by up to 43 points. On two production-style deployments built from public benchmark tasks, it raises intent classification from 84.9% to 99.3% and Entity F1 from 0.345 to 0.810. Beyond performance gains, the agent often discovers effective training strategies, including chain-of-thought supervision, task-specific optimization, and quality-focused data curation, purely from downstream feedback.

### 2. 方法论：Pioneer Agent 系统
Pioneer Agent 是一个基于 LangGraph 状态机、由 Claude 4.6 Sonnet 编排的自动化系统，包含两种运行模式：
*   **冷启动模式 (Cold-Start Mode)**：
    *   **输入**：仅需自然语言任务描述。
    *   **流程**：自主进行任务研究、下载/合成数据、构建验证集，并通过 **蒙特卡洛图搜索 (MCGS)** 联合优化数据集 ($D$)、超参数 ($H$) 和学习策略 ($S$)。
*   **生产模式 (Production Mode)**：
    *   **输入**：已部署模型及其带标注的推理失败日志。
    *   **流程**：构建故障分类法（Taxonomy）、通过针对性探测确认弱点、合成包含“黄金修正+硬负样本+重放数据”的课程、在显式回归约束下重训。
*   **关键技术细节**：
    *   **MCGS 算法**：维护一个实验图，记录不同训练尝试的因果关系，支持分支融合（Fusion）和进化（Evolution）。
    *   **回归门控 (Regression Gating)**：强制要求新模型在修复故障的同时，在旧有的正确样本上回归数量不得超过 2 个（$\epsilon=2$）。
    *   **数据策展原则**：遵循“2换1”规则（1个正例配1个相似负例）、标签平衡和上下文长度匹配。

*   **实验规模**：论文涵盖了 8 个冷启动任务、7 个 AdaptFT-Bench 场景以及 2 个深度生产案例，总计进行了数十次完整的迭代搜索。
*   **充分性评价**：实验设计较为全面，不仅对比了最终得分，还详细记录了迭代过程中的性能曲线（如分支融合的效果、回滚机制的触发）。通过合成噪声模拟生产漂移，增强了实验的客观性和说服力。

### 6. 主要结论与发现
*   **性能提升**：在冷启动任务中，性能提升了 1.6 到 83.8 个百分点。在生产模式下，Pioneer Agent 成功规避了朴素重训导致的性能崩溃（最高可跌 43 分）。
*   **自主发现策略**：智能体在没有明确指令的情况下，自主发现了**思维链 (CoT) 监督**、**特定任务的 Epoch 敏感性**以及**系统提示词工程**等高级微调技巧。
*   **数据质量胜过数量**：实验证明，少量高质量、经过诊断的样本比大规模带噪数据更有效。

### 7. 优点
*   **全生命周期自动化**：实现了从任务定义到生产修复的闭环，减少了对资深算法工程师的依赖。
*   **安全性保障**：引入了严格的回归测试和回滚机制，这在实际生产部署中至关重要。
*   **策略发现能力**：MCGS 搜索不仅优化了参数，还探索了数据表示和学习逻辑。

### 8. 不足与局限
*   **模型支持有限**：目前仅支持 GLiNER2、Qwen 和 Llama 系列，尚未扩展到视觉模型或更复杂的对齐算法（如 RLHF/DPO）。
*   **评估依赖**：系统高度依赖“LLM-as-Judge”或人工标注的质量，如果反馈信号本身有偏，模型改进可能会受限。
*   **智能体开销**：编排智能体（如 Claude 4.6）本身会产生非琐碎的 Token 成本和延迟，对于极简单的微调任务可能显得“大材小用”。

## 摘要
大语言模型（LLM）智能体的强化学习通常在静态数据分布上进行，这无法适应智能体不断演化的行为，并导致对复杂环境交互的覆盖不足。为了应对这些挑战，我们提出了 CoEvolve，这是一个智能体-数据协同演化框架，使 LLM 智能体能够通过闭环、交互驱动的训练实现自我提升。具体而言，CoEvolve 从展开轨迹中提取遗忘和不确定性等反馈信号，以识别易出错的交互模式，并利用这些信号指导基于 LLM 的任务合成。合成的任务通过环境交互进行验证，并用于更新数据分布，从而实现智能体及其数据的共同适应。在 AppWorld 和 BFCL 基准测试上，针对 Qwen2.5-7B、Qwen3-4B 和 Qwen3-30B-A3B 进行的广泛实验表明，该方法相较于强大的基础模型取得了持续且显著的提升，绝对增益分别达到 19.43%、15.58% 和 18.14%。

## Abstract
Reinforcement learning for LLM agents is typically conducted on a static data distribution, which fails to adapt to the agent's evolving behavior and leads to poor coverage of complex environment interactions. To address these challenges, we propose CoEvolve, an agent-data mutual evolution framework that enables LLM agents to improve through closed-loop, interaction-driven training. Specifically, CoEvolve extracts feedback signals such as forgetting and uncertainty from rollout trajectories to identify failure-prone interaction patterns, and utilizes them to guide LLM-based task synthesis. The synthesized tasks are validated through environment interaction and utilized to update the data distribution, enabling joint adaptation of the agent and its data. Extensive experiments on AppWorld and BFCL across Qwen2.5-7B, Qwen3-4B, and Qwen3-30B-A3B demonstrate consistent and significant improvements over strong base models, yielding absolute gains of 19.43%, 15.58%, and 18.14%, respectively.

*   **整体含义**：论文提出了 **CoEvolve** 框架，通过“智能体-数据协同演化”的闭环机制，让训练数据随智能体的进步而进化，从而实现无须人工干预的自我提升。

### 2. 方法论：核心思想与关键技术
CoEvolve 的核心思想是利用智能体训练过程中的反馈信号来指导新任务的合成。其流程分为三个阶段：
*   **阶段 1：训练与信号提取**
    *   使用 **GRPO**（群体相对策略优化）算法进行策略训练。
    *   从训练轨迹中提取三种**弱点信号**：
        1.  **遗忘信号 (Forgetting Signals)**：检测智能体曾经能完成但当前策略下失败的任务。
        2.  **边界信号 (Boundary Signals)**：识别在同一任务中表现不稳定的情况（即多次采样中既有成功也有失败）。
        3.  **稀有信号 (Rare Signals)**：识别在训练中出现频率极低但具有系统性的动作模式。
*   **阶段 2：信号引导的环境再探索**
    *   利用 LLM 对带有信号标注的失败轨迹进行反思，构建结构化的探索上下文。
    *   引导 LLM 在环境中进行多轮、多步的再探索，寻找能够克服弱点的替代行为。
*   **阶段 3：任务抽象与验证**
    *   将探索得到的交互三元组（动作-观察-标识）抽象为新的任务描述和解决方案。
    *   在真实环境中验证这些合成任务的有效性，通过验证的任务被加入训练集，更新数据分布。

### 5. 实验数量与充分性
*   **实验规模**：涵盖了从 4B 到 30B 不同规模的三个骨干模型，并在多个具有挑战性的交互式基准上进行了测试。
*   **消融实验**：非常充分。作者分别验证了三种反馈信号（遗忘、边界、稀有）的各自贡献、任务验证环节的必要性、合成数据的多样性与相关性、以及不同探索模型质量对结果的影响。

### 6. 主要结论与发现
*   **显著提升**：CoEvolve 在 Qwen2.5-7B、Qwen3-4B 和 Qwen3-30B 上分别实现了 **19.43%、15.58% 和 18.14%** 的平均性能提升。
*   **超越闭源模型**：经过 CoEvolve 训练的 Qwen3-4B 在 BFCL 上的表现（63.00）超过了 GPT-4（54.00）和 Gemini-2.5-Flash（41.50）。
*   **协同效应**：反馈信号引导的任务合成比随机探索更有效，且遗忘信号对防止模型性能退化至关重要。

### 7. 优点
*   **闭环自进化**：实现了智能体与数据的双向优化，摆脱了对高质量人工标注轨迹的依赖。
*   **精准打击弱点**：通过遗忘和边界信号，训练过程能自动聚焦于模型的薄弱环节，提高训练效率。
*   **高扩展性**：框架不依赖特定模型，且计算开销增加较小，易于在大规模模型训练中推广。

### 8. 不足与局限
*   **早期信号噪声**：在训练初期，智能体能力尚不成熟，提取的反馈信号可能存在噪声，影响任务合成的质量。
*   **安全与合规风险**：由于是自主合成任务并更新分布，在涉及真实环境或安全敏感的场景下，可能需要额外的安全过滤机制（目前主要依赖环境验证）。
*   **信号类型有限**：目前仅探索了三种信号，可能还存在其他更具启发性的反馈信号（如推理链的逻辑一致性等）尚未被利用。

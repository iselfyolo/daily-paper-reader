Abstract

We formalize this as Controlling Authority Retrieval (CAR) :recovering the active frontier front(cl( Ak(q))) 

of the authority closure of the semantic anchor set—a different mathematical problem from 

arg max d s(q, d ).The two central results are: Theorem 4 (CAR-Correctness Characterization) gives necessary-and-sufficient conditions on any retrieved set

1 Introduction 

In any domain where knowledge accumulates in-crementally, later facts can formally override ear-lier ones. Legal precedents are overruled by later courts. FDA drug approvals are superseded by re-call enforcement actions. Security patches void prior vulnerability disclosures. Compliance clear-ances are overridden by blackout announcements. In each case, the correct answer to a query does not depend on the most semantically relevant document alone—it depends on whether that document has been superseded by a later one. We call this family of retrieval problems controlling authority retrieval ,and show that standard retrieval systems fail on it structurally across all three authority-governed do-mains we test. Retrieval-augmented generation (RAG) (Lewis et al., 2020) ranks documents by semantic rele-vance to the query. This is silent on supersession. A RAG system asked whether a drug approval is still valid retrieves the approval document (highest relevance), not the recall notice (low relevance, dif-ferent vocabulary). It has no mechanism to detect that the approval was voided. 

Running example. A security engineer asks: “Is the prototype pollution vulnerability in lodash still unpatched?” The most relevant document is the CVE disclosure—it matches lodash , prototype pol-

2.1 Temporally Ordered Knowledge Bases Definition 1 (Document and Supersession Rela-tion) . A temporally ordered knowledge base K =

Definition 2 (Controlling Authority Retrieval) . Aretrieval task is a controlling authority task if the correct document to return is not the most semanti-cally relevant document in the corpus, but rather the document that controls the semantically rele-vant one under a formal precedence or superses-sion relation. Controlling authority tasks form a proper subset of retrieval tasks, characterized by three properties: (i) There exists an authoritative ordering ⇝ on documents independent of semantic similarity to the query. ( ⇝ is any strict partial order; temporal precedence is one instance, but au-thority can be non-temporal, e.g., statutory hierarchy over case law, or ontological sub-sumption.) (ii) The query vocabulary aligns with a non-controlling document (the anchor ). (iii) The controlling document has domain-distinct vocabulary (different event type, action terms, or register). 

Definition 3 (Authority Closure and Active Fron-tier) . Let A ⊆ K be a set of documents and ⇝ the authority order on K. The authority closure of A

is: 

cl( A) = A ∪ { d′ ∈ K : ∃ d ∈ A, d ⇝ d′}

The active frontier of a set S ⊆ K is the set of maximal (non-superseded) elements: 

front( S) = {d ∈ S : ∄ d′ ∈ S s.t. d ⇝ d′}

The operators cl( ·) and front( ·) are standard con-structions in order theory (closure under a partial order; antichain of maximal elements); the con-tribution is identifying that standard retrieval opti-mizes neither, and characterizing the resulting gap. Remark 1 (Generality of TCA) . TCA requires only a supersession relation ⇝ and a gold answer func-tion. It applies to any domain where documents have a formal validity lifecycle—legal, medical, regulatory, or operational. 

2.3 Four TC-MQA Hop Types T0 (No supersession): One active document an-swers the query. Equivalent to standard single-hop retrieval. 

T1 (2-hop direct supersession): d1 ⇝ d2; the query refers to d1’s subject. The correct answer requires d2, which may be semantically distant from Q.

T2 (3-hop chain): d1 ⇝ d2 ⇝ d3; the full chain must be available for TCA = 1. 

T3 (3-hop cross-provenance): The superseding document d3 originates from a different document source (session, jurisdiction, team) than d1 and 

d2, while sharing entity scope. This tests retrieval across provenance boundaries. Query templates for all four hop types are given in Appendix B.

exception), drug regulation (approval × recall ×

front(cl( Ak(q))) . The streaming CAR problem is to maintain front(cl( Ak(q))) under insertions in sublinear amortized time per query. Each insertion invalidates at most |front | cached retrievals (those superseded by the new document), giving a natural incremental update structure.

4 FIN SUPER QA: A TC-MQA Benchmark 

FIN SUPER QA instantiates TC-MQA in SEC Rule 204A-1 employee trading compliance (U.S. Securi-ties and Exchange Commission, 2004). The entity scope is σ = {(emp id , ticker )}; supersession fol-lows rules R1–R10 (Appendix A). Ground-truth answers are algorithmically derivable from active events, eliminating annotation ambiguity. Each of the 1,000 examples is assigned a unique (employee, ticker) entity pair from a pool of 6,000. This entity-disjoint design ensures the entity probe retrieves each example’s gold events cleanly. The shared corpus contains 12,250 events (2,250 gold + 10 distractors per example, using three distractor types: same-employee/different-ticker, different-employee/same-ticker, and random). A contami-nated variant relaxes the disjoint guarantee to study disambiguation (§6.3). 5 Systems 

Baselines. BM25 (Robertson and Walker, 1994) (k1 = 1 .5, b = 0 .75 ) and TF-IDF (cosine) are the lexical baselines. Dense (MiniLM) uses sentence-transformer embeddings (Reimers and Gurevych, 2019). Oracle ranks gold documents first (upper bound). Full hyperparameters and graph construc-tion details are in Appendix C. 

DENSE RERANK : dense retrieval with supers-ession re-ranking. Promotes superseders of top-20 events that are superseded in the gold KB; restricted to re-ranking what dense retrieval already surfaced. This is the strongest re-ranking baseline: it has access to ground-truth supersession edges. 

TWO STAGE (anchor probe): dense retrieval + entity-indexed probing. Extracts entity iden-tifiers from the query; for every entity-matched event in the initial top-20, looks up the full-corpus entity index for later-timestamped events in the same scope and promotes them. This implements the Decomposition Theorem’s three stages: an-chor discovery (dense top-20), authority resolu-tion ( cl( A) via entity index), active-status deter-mination ( front(cl( A)) by timestamp). Achieves TCA = 1 .000 on entity-disjoint corpora. 

TWO STAGE +RSSG (ours): T WO STAGE aug-mented with a Retrieved-Set Supersession Graph (RSSG) . After the entity probe, applies domain rules R1–R10 (Appendix A) to all retrieved entity-scoped events—without gold KB access—to deter-mine which events are active under the superses-sion relation. This implements front(cl( A)) cor-rectly when κ(q) > 1: the RSSG identifies which retrieved documents are in the true cl( A) vs. spuri-ous same-scope distractors, enabling accurate fron-tier recovery ( +12 .4 pp on T3).

6.6 Two-Stage Pipeline for Free-Form Queries 

The Regex-NER Probe achieves TCA = 1 .000 on the structured query template “Is {product } still affected by {CVE-ID }?”, but this template encodes entity identity explicitly. In practice, developers ask free-form questions—“Has the issue with path parameters being substituted into URLs without encoding been fixed?”—that describe vulnerability behavior without the CVE identifier. The entity probe cannot be applied to such queries. 

Free-form queries. We generate one free-form question per GHSA pair using Llama 3.3 70B (Nebius). Each prompt provides the disclosure text and instructs the model to write a natural-language question (maximum 50 words) describing the vul-nerability behavior, without including the CVE identifier and without using the structured template. All 159 generated queries were verified to contain no CVE-ID string (regex CVE-\d{4}-\d+ ). 

Key observation. BM25 on a free-form vulner-ability question finds the CVE disclosure reliably (recall@1 = 91.8%, @5 = 97.5%) even though it cannot find the patch note (TCA@5 = 0.138). Both the free-form question and the disclosure describe the same vulnerability in similar vocabulary; the gap is specifically between the query vocabulary and the patch-note vocabulary, not between the query and the disclosure. 

Two-stage pipeline. We exploit this asymmetry via a two-stage architecture: 1. Stage 1 (disclosure retrieval): BM25 retrieves the top-k1 documents from the full corpus using the free-form query. 2. Stage 2 (entity-indexed patch lookup): For each Stage-1 result that is a disclosure, use its CVE-ID metadata as a key into the entity index to retrieve the paired patch note. The final ranking places Stage-2 patch candidates first (in Stage-1 order), followed by remaining doc-uments.

6.11 Downstream LLM Answer Quality: Dense vs. Two-Stage 

TCA measures retrieval correctness, but the ulti-mate risk is downstream: a language model given superseded evidence will produce confident, incor-rect answers that a practitioner may act on. We test this directly by running GPT-4o-mini on top of Dense and Two-Stage retrievers on the 159 GHSA pairs. 

Setup. Each query is issued against a 2,318-document corpus. Dense retrieval returns MiniLM top-5; Two-Stage returns BM25 Stage-1 top-5, entity-indexed to promote the patch note. GPT-4o-mini is prompted to answer: “Is the vulnerability patched? If so, what version fixes it?” using only the retrieved documents. The gold answer for every pair is yes, patched (all pairs have a real patch note). We classify each response as: correct (explicitly states patched and names the fixing version), con-fident wrong (explicitly asserts the vulnerability is not patched), or uncertain (hedged or no patch status claimed). 

Results. Table 12 shows the outcome. Dense re-trieval correctly identifies the patch for only 61.0% of queries; for 39.0% of queries (95% CI: 31.3– 47.1%), GPT-4o-mini explicitly asserts the vul-nerability remains unpatched—a confident, action-able wrong answer despite every gold answer be-ing patched . The gap is explained by retrieval: Dense retrieves the patch note for only 28.3% of queries; when it misses (71.7% of queries), GPT-4o-mini produces a confi-dent “not patched” assertion at a 54% rate. 7 Related Work 

Multi-hop QA. HotpotQA (Yang et al., 2018), MuSiQue (Trivedi et al., 2022), and 2WikiMulti-Hop (Ho et al., 2020) require multi-step retrieval. IRCoT (Trivedi et al., 2023), PropRAG (Tang et al., 2025), and HippoRAG 2 (Guti ´errez et al., 2025) im-prove multi-hop retrieval through chain-of-thought interleaving, graph propagation, and associative memory. These systems assume all required doc-uments are retrievable by relevance. TC-MQA breaks this assumption: the superseding document may be semantically orthogonal to the query (The-orem 1), making relevance improvement insuffi-cient. 

Temporal RAG. T-GRAG (Li et al., 2025) re-solves temporal redundancy in graph RAG. Ver-sionRAG (Huwiler et al., 2025) handles version-aware retrieval for evolving documents. Tem-pRALM (Gade and Jetcheva, 2024) incorporates timestamps into retrieval scoring. LedgerRAG (Wang et al., 2025) provides governance-aware agentic retrieval. These works address recency preference —favoring newer documents—whereas TC-MQA requires detecting that a specific later document formally voids an earlier one under do-main rules. The difference is structural: recency preference is a soft ranking adjustment; superses-sion is a hard validity constraint. Importantly, CAR is not equivalent to time-aware retrieval: the au-thority relation ⇝ is not timestamp ordering—a superseding document can predate its superseded counterpart when effective dates differ from publi-cation dates (common in regulatory and legal cor-pora), so timestamp-conditioned ranking cannot recover front(cl( Ak(q))) in general. 

Legal and regulatory temporal reasoning. 

Chalkidis et al. (Chalkidis et al., 2021) provide the earliest NLP recognition that temporal filtering is essential for regulatory compliance IR, showing that BM25 with chronological constraints outper-forms pure neural re-rankers on EU/UK directive retrieval—an empirical precursor to Theorem 1’s mismatch result. Zhang et al. (Zhang et al., 2025) find that LLMs backed by standard RAG consis-tently fail to identify when legal precedents are overruled—a direct real-world instantiation of The-orem 1. Stammbach et al. (Stammbach et al., 2026) document that existing legal retrieval benchmarks fail to transfer to practitioner contexts, underscor-ing the need for domain-specific retrieval objec-tives. Di Florio et al. (Di Florio et al., 2024) formal-ize legal conflict resolution with temporal ordering, providing formal apparatus complementary to our empirical framework. CaseFacts (Putta et al., 2026) benchmarks precedent retrieval. RIRAG (Gokhan et al., 2024) and FinDER (Choi et al., 2025) address regulatory and financial QA without supersession structure. 

Supersession-aware retrieval. The closest prior work is de Martim (de Martim, 2025), who pro-poses SAT-Graph RAG—an ontology-driven graph RAG that models legislative events as queryable nodes and handles point-in-time legal retrieval. Like CAR, SAT-Graph RAG recognizes that flat-text retrieval is blind to the diachronic structure of law and uses a graph to represent norm super-session. The key distinction is theoretical: SAT-Graph RAG is an engineering system for struc-tured legal corpora with no formal retrieval objec-tive, no correctness characterization (Theorem 4), and no hardness result showing that scope-indexed algorithms face an identifiable worst-case ceiling (Proposition 2). CAR defines front(cl( Ak(q))) as an independent retrieval objective applicable to any authority-governed domain, and proves necessary-and-sufficient conditions on any retrieved set inde-pendent of how it was produced. 

Temporal knowledge graphs. Temporal KG QA (Su et al., 2024) and TDBench (Kim et al., 2026) ad-dress time-sensitive graph queries over structured KGs. TC-MQA is the unstructured-text analogue: supersession must be inferred from document type and timestamps rather than read from graph edges. 

RAG limitations. Weller et al. (Weller et al., 2026) prove theoretical limits of embedding-based retrieval; Theorem 1 is a domain-specific instantia-tion with an explicit supersession construction. Bar-nett et al. (Barnett et al., 2024) catalog seven RAG failure modes; Controlling Authority Retrieval is a distinct eighth mode not in their taxonomy— one where the correct answer is provably unreach-able by relevance ranking. Rashtchian and Juan (Rashtchian and Juan, 2025) study retrieval suf-ficiency; TCA formalizes a stronger sufficiency notion that includes temporal validity. 

8 Research Program 

Formalizing CAR opens several research direc-tions, each corresponding to a relaxation of one component of front(cl( Ak(q))) . The four general-ized CAR variants are defined formally in §2.4; the directions below elaborate their open problems. 

Latent-scope CAR (Definition 8). When scope is unobserved, the retrieval problem becomes in-ference over p(σ | q). The key open problem: given a query-time NER error rate ϵ, what is the achievable TCA ceiling? Proposition 2 gives a 

φ(q)-parameterized upper bound; the latent-scope case requires integrating over the scope posterior. 

Probabilistic-authority CAR (Definition 9). Learning π(d, d ′) from text is a new structured pre-diction problem: the target relation is “B formally voids A under domain rules,” not “A happened before B.” Risk-sensitive frontier recovery under uncertain edges connects to distributionally-robust optimization. 

Multi-authority CAR (Definition 10). The lat-tice of active frontiers under m interacting orders is an open combinatorial problem. For m = 2 , fron-tier recovery generalizes to finding the Pareto front under two partial orders; for m ≥ 3, this is com-putationally hard in general (multi-dimensional Pareto dominance). 

Streaming CAR (Definition 11). Maintain-ing front(cl( Ak(q))) under document arrivals has amortized update cost O(|front |) per insertion— but the frontier can change discontinuously when a new document supersedes multiple existing ones. Batch-invalidation and lazy-update strategies are unexplored. 

Authority-preserving embeddings. Train representations so that cl( A) is geometrically identifiable—anchors and their controlling author-ities cluster in authority-closure space rather than semantic space. The new contrastive objective: 

d ⇝ d′ should bring d and d′ close in embedding space without conflating their semantic roles. 

Budgeted closure recovery. Given lookup bud-get B, maximize |front(cl( Ak(q))) | subject to B.Proposition 3 gives the lower bound on necessary lookups; the gap between the lower bound and prac-tical budget is the open problem. This connects to adaptive submodular maximization. 

Normative retrieval. CAR is one instance of the broader class where the retrieval target is the normatively correct document under a rule system, not the most relevant one. Characterizing the com-plexity of this class—which problems reduce to CAR, which require stronger primitives—is the natural theoretical generalization. 

9 Limitations 

Entity extraction in free-form text. The FinSu-perQA benchmark uses structured employee IDs that admit 100% regex extraction, and the GHSA evaluation with structured templates admits 100%-accurate Regex-NER. For free-form queries with-out the CVE identifier, the two-stage pipeline (§6.6) achieves TCA@5 = 0 .975 by using BM25 to find the CVE disclosure first, then mapping disclosure metadata to the paired patch note via entity-indexed lookup. The 2.5% failure rate arises from three generic queries with insufficient vocabulary over-lap and one package with no BM25 term match; in these cases the pipeline degrades to the BM25 baseline. In fully free-form settings where product names are absent or misspelled, additional NER or entity linking would be required. 

Entity collision in production corpora. On an entity-disjoint corpus—where each (employee, security) pair appears in exactly one example— anchor-based probing fully solves T0–T3 (TCA =1.000 ). In corpora where entities recur across cases (e.g., an employee with multiple concurrent open positions), the entity probe surfaces events from all cases, requiring additional disambiguation. The contamination ablation (§6.3) shows RSSG recov-ers +12 .4 pp in this regime ( 0.652 → 0.776 ); clos-ing the full gap requires case-boundary metadata or temporal windowing not always available in prac-tice. 

Synthetic supersession rules. The 10 rules cover representative compliance scenarios. Production deployment would require domain-expert rule spec-ification for a specific regulatory framework. 

Evaluated but absent from prior work. The cross-encoder reranker and OSV.dev (the security production lookup tool) are now evaluated (§6.6). The cross-encoder (TCA@5 = 0 .233 ) confirms that reranking within a relevance-ranked pool can-not substitute for entity-indexed Stage-2 retrieval. OSV.dev is 0% applicable under free-form queries— it requires an explicit CVE identifier and solves a different problem from CAR. Shepard’s Citations (legal supersession) was not evaluated; it requires a subscription not available during this study. The remaining absent baseline is fine-tuned bi-encoders 

trained on domain-specific (q, d ∗) pairs. Remark 2 argues that such models implement proactive en-tity search implicitly and are therefore counterex-amples in form only; fine-tuning on 127 GHSA pairs (3 epochs, MultipleNegativesRankingLoss) produced a model with TCA@5 = 0 .438 —below the zero-shot MiniLM baseline ( 0.500 ) and 53 pp below Two-Stage (0 .969) . The fine-tuning run did not improve in-domain performance, confirming that the gap is architectural rather than a training-data artifact. 

Scope of current theory. Theorems 1–4 and Propositions 1–3 characterize static, deterministic, single-authority CAR with recoverable scope . The four generalized variants (Definitions 8–11) for-malize the extensions: latent-scope, probabilistic-authority, multi-authority, and streaming CAR. Each corresponds to relaxing exactly one compo-nent of front(cl( Ak(q))) and maps to a specific term in Proposition 1—scope posterior for φ(q),edge confidence for ⇝, order count for the lattice structure, and corpus dynamics for Ranchor .

10 Conclusion 

The paper makes two primary theoretical claims about the class of authority-governed retrieval prob-lems, independent of any particular system. 

Theorem 4 gives the necessary-and-sufficient conditions on any retrieved set R for TCA = 1 :frontier inclusion ( front(cl( Ak(q))) ⊆ R) and no-ignored-superseder. Any future algorithm— learned, hybrid, or rule-based—can be audited against these two conditions without reference to how it was built. 

Proposition 2 establishes that scope-indexed al-gorithms (those without access to the authority rela-tion ⇝) face a hard worst-case ceiling: TCA@ k ≤φ(q) · Ranchor (q). The proof is adversarial, not average-case: when κ(q) authority chains share entity scope, no algorithm that cannot distinguish them achieves more than 1/κ (q) = φ(q) condi-tional success probability. These two results are supported by three theo-rems that characterize the problem structure: Theo-rem 1 (TCA and Recall@ k decouple when ∆( q) >

0), Theorem 2 (authority closure is necessary for TCA = 1 ), and Theorem 3 (anchor discovery and authority resolution are each necessary; Corollary 3 proves TCA = 1 is achievable when φ(q) = 1 ). Proposition 1 organizes the picture into the fac-torization TCA = Ranchor · φ · Rfrontier , whose three terms are separable failure modes (Lemma 1). Proposition 3 establishes the computational cost of disambiguation under an explicit oracle model. Three independent real-world corpora con-firm that the proved structure is consequential: Dense TCA@5 ∈ {0.064 , 0.172 , 0.270 } across FDA, SCOTUS, and GHSA; Two-Stage achieves 

{0.774 , 0.926 , 0.975 }. The 77.4% FDA ceiling is the corpus-level average of φ(q) at Stage-1— a property of the scope distribution, not a retrieval limitation. A GPT-4o-mini downstream experiment quantifies the cost of ignoring supersession: confi-dent “not patched” claims for 39% of queries with a patch; Two-Stage reduces this to 16%. The four formal generalizations (Definitions 8– 11) identify the next layer of open problems: latent-scope CAR, probabilistic-authority CAR, multi-authority CAR, and streaming CAR. Each is a pre-cisely stated problem class, not a vague direction, and each inherits Theorem 4 as the correctness tar-get.

D Domain Adapters and Scorer Interface 

Adapting the two-stage pipeline to a new authority domain requires three attributes: • anchor type : document type that is se-mantically aligned to the query (e.g., CVE 

> DISCLOSURE

, original SCOTUS opinion, FDA approved label). • superseding type : document type that formally voids the anchor (e.g., PATCH RE -

> LEASE NOTE

, overruling opinion, FDA recall enforcement action). • scope keys : entity fields that co-scope anchor and superseder (e.g., 

[cve id, package] , [case slug] ,

[ndc code] ). Three domain adapters are re-leased alongside this paper: security 

(realworld/ghsa twostage eval.py ), 

legal (realworld/legal lii benchmark.py ), and medical (realworld/fda recall eval.py ). Each adapter is a self-contained Python script that fetches or loads its domain dataset, builds the entity index, runs all baselines, and writes results to data/ .

Scorer interface. evaluate.py runs the full synthetic FinSuperQA benchmark. The custom re-triever class must implement retrieve(self, query: str, corpus: list[dict], k: int) -> list[str] , where corpus 

is the list of document dicts and the return value is a list of doc id strings. The scorer computes TCA, Acc, ProvRec, and R@ k on all 1,000 examples with a single call: 

python evaluate.py --retriever_path my_retriever.py --k 5 

Output is written to 

data/custom retriever results.json .All datasets, domain adapters, evaluation scripts, and the falsification suite are available at https://github.com/andremir/ car-retrieval .

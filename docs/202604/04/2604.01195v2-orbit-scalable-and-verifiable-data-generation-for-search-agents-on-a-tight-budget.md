## 摘要
搜索智能体将语言模型（LM）与网络搜索相结合，在回答复杂用户查询方面正变得至关重要。由于昂贵的人工标注或繁琐的前提条件，构建涉及多步检索和推理的深度研究任务训练数据集仍然具有挑战性。在这项工作中，我们介绍了 ORBIT，这是一个包含 2 万个推理密集型查询及其简短可验证答案的训练数据集，该数据集是使用一种不依赖付费 API 服务的节俭框架生成的。该模块化框架包含四个阶段：种子创建、问答对生成，以及两个验证阶段（自我验证和外部验证）。ORBIT 涵盖 15 个领域，每个训练对需要 4-5 个推理步骤，并需要通过全网进行外部搜索验证。我们使用 GRPO 算法在 ORBIT 上训练 Qwen3-4B 作为基础模型，并在维基百科问答任务上进行了评估。广泛的实验结果表明，ORBIT-4B 在 4B 参数以下的搜索智能体大语言模型（LLM）中表现强劲，证明了合成数据集的效用。我们的框架、代码和数据集已开源并公开发布。

## Abstract
Search agents, which integrate language models (LMs) with web search, are becoming crucial for answering complex user queries. Constructing training datasets for deep research tasks, involving multi-step retrieval and reasoning, remains challenging due to expensive human annotation, or cumbersome prerequisites. In this work, we introduce ORBIT, a training dataset with 20K reasoning-intensive queries with short verifiable answers, generated using a frugal framework without relying on paid API services. The modular framework relies on four stages: seed creation, question-answer pair generation, and two stages of verification: self and external. ORBIT spans 15 domains and each training pair requires 4-5 reasoning steps, with external search verification required from the complete web. We train Qwen3-4B as the base model on ORBIT using GRPO and evaluate it on Wikipedia question answering tasks. Extensive experiment results demonstrate that ORBIT-4B achieves strong performance among sub-4B LLMs as search agents, proving the utility of synthetic datasets. Our framework, code and datasets are open-sourced and available publicly.

*   **研究动机**：现有的训练数据集（如 NQ, HotpotQA）通常过于简单（仅需1-2步搜索），而高质量的复杂推理数据标注成本极高，且往往依赖昂贵的付费 API（如 GPT-4 或 Google Search API）或复杂的前置条件（如构建知识图谱）。
*   **核心目标**：在**极低预算**（Tight Budget）下，构建一个大规模（20K）、高难度、且答案可验证的合成数据集，以提升轻量级模型（<4B参数）作为搜索智能体的推理能力。

### 2. 方法论：核心思想与关键技术
ORBIT 提出了一个**四阶段的节俭型（Frugal）数据生成框架**，完全不依赖付费 API：

1.  **种子创建（Seed Creation）**：手动确定 15 个领域（如科技、医学、法律、谜题等），利用维基百科类别（Categories）和页面标题作为初始种子，确保覆盖长尾知识。
2.  **问答对生成（QA Pair Generation）**：使用 Python Selenium 自动化操作 **DeepSeek-V3.1 网页版**（开启 DeepThink 和搜索功能）。给定种子，生成“倒置问题”（即描述复杂事实但指向简短答案的问题），要求每个问题至少需要 5 个证据支撑。
3.  **自我验证（Self-Verification）**：再次利用 DeepSeek-V3.1 网页版对生成的 QA 对进行验证。模型需进行全网搜索，提供验证声明并引用 URL。通过 Qwen3-4B 过滤掉验证不充分的样本。
4.  **外部验证（External Verification）**：为避免模型自我偏见，使用级联的 LLM 评判器（Qwen3-4B-Instruct 和 GPT-OSS-120B）。系统抓取验证阶段提供的 URL 内容，让评判器根据网页原文独立回答问题，并与原始答案比对，仅保留一致的样本。

*   **训练算法**：**GRPO**（Group Relative Policy Optimization），这是一种无需批判者（Critic）模型的强化学习算法，通过精确匹配（Exact Match）作为奖励信号。
*   **数据集与 Benchmark**：

    *   **多跳 QA**：HotpotQA (HQA), 2WikiMultihopQA, MuSiQue, Bamboogle。

### 6. 主要结论与发现
*   **性能领先**：ORBIT-4B 在所有单跳和多跳基准测试中均显著优于现有 4B 以下模型。在 Wikipedia 综合任务上，比 Search-R1-4B 高出 **9.0 EM 分**，比 InfoSeeker-4B 高出 **4.3 EM 分**。

*   **小模型的潜力**：通过 GRPO 强化学习，4B 规模的小模型也能展现出极强的多步搜索计划和信息聚合能力。

### 7. 优点（亮点）
*   **极低成本**：通过自动化网页端交互绕过了高昂的 API 费用，为预算有限的实验室提供了可行方案。

*   **完全开源**：开源了框架、20K 验证数据集、代码和训练好的模型。

## 摘要
大语言模型（LLMs）显著推动了工具增强型智能体的发展，使其能够通过 API 交互实现自主推理。然而，在海量工具库中执行多步任务仍然面临挑战，主要归因于两个关键瓶颈：（1）缺乏严格的计划级评估框架；（2）大规模工具集和长程规划带来的庞大决策空间探索所产生的计算需求。为了填补这些空白，我们首先引入了 SLATE（电子商务合成大规模 API 工具包），这是一个专为工具集成智能体自动化评估而设计的大规模上下文感知基准测试。与静态指标不同，SLATE 能够容纳多样化且功能有效的执行轨迹，揭示了当前智能体在自我修正和搜索效率方面存在困难。受这些发现的启发，我们随后提出了熵引导分支（EGB），这是一种不确定性感知搜索算法，能够在预测熵较高的位置动态扩展决策分支。EGB 优化了探索与利用之间的权衡，显著提高了任务成功率和计算效率。在 SLATE 上的广泛实验表明，我们的双重贡献为在工具丰富的环境中开发可靠且可扩展的 LLM 智能体奠定了坚实基础。

## Abstract
Large Language Models (LLMs) have significantly advanced tool-augmented agents, enabling autonomous reasoning via API interactions. However, executing multi-step tasks within massive tool libraries remains challenging due to two critical bottlenecks: (1) the absence of rigorous, plan-level evaluation frameworks and (2) the computational demand of exploring vast decision spaces stemming from large toolsets and long-horizon planning. To bridge these gaps, we first introduce SLATE (Synthetic Large-scale API Toolkit for E-commerce), a large-scale context-aware benchmark designed for the automated assessment of tool-integrated agents. Unlike static metrics, SLATE accommodates diverse yet functionally valid execution trajectories, revealing that current agents struggle with self-correction and search efficiency. Motivated by these findings, we next propose Entropy-Guided Branching (EGB), an uncertainty-aware search algorithm that dynamically expands decision branches where predictive entropy is high. EGB optimizes the exploration-exploitation trade-off, significantly enhancing both task success rates and computational efficiency. Extensive experiments on SLATE demonstrate that our dual contribution provides a robust foundation for developing reliable and scalable LLM agents in tool-rich environments.

#### 1. 核心问题与整体含义（研究动机和背景）
随着大语言模型（LLM）的发展，工具增强型智能体在自动化复杂工作流方面展现了巨大潜力。然而，在面对**大规模工具库（1000+ API）**和**长程规划（10步以上）**的任务时，现有智能体面临两大瓶颈：
*   **评估框架缺失**：现有的基准测试要么工具规模太小，要么缺乏确定性的执行反馈，往往依赖主观的“LLM-as-a-judge”，无法准确衡量智能体在复杂路径下的端到端成功率。
*   **搜索效率低下**：现有的搜索算法（如 MCTS）在长程任务中计算成本极高，且难以在庞大的决策空间中有效平衡探索与利用。

#### 2. 方法论：核心思想与算法流程
本文提出了双重解决方案：**SLATE 基准测试**和 **EGB 搜索算法**。

*   **SLATE (Synthetic Large-scale API Toolkit for E-commerce)**：
    *   一个包含 1000 个工具的电商领域基准测试，支持长程执行（平均 10+ 步）。
    *   **核心技术**：引入了确定性模拟器（Simulator），为每个工具调用提供上下文感知的反馈，支持“计划级落地评估”（Plan-Level Grounded Evaluation），即只要最终状态正确，允许智能体采取与参考路径不同的有效轨迹。
*   **EGB (Entropy-Guided Branching, 熵引导分支)**：
    *   **阶段 1：初始探索与不确定性量化**。智能体按计划执行，在每一步通过多次采样（黑盒模式）或提取 Logits（白盒模式）计算预测熵（Entropy），以此衡量工具选择的不确定性。
    *   **阶段 2：排序分支与迭代搜索**。如果初始执行失败，算法会根据记录的熵值对步骤进行降序排列，优先在熵值最高（即模型最不确定）的步骤处开启新分支，尝试备选工具并重新执行后续步骤。
    *   **优势**：相比 MCTS 的全局搜索，EGB 是一种局部诊断式搜索，能精准定位并修复导致失败的关键决策点。

*   **数据集/场景**：

*   **实验规模**：在 SLATE 和 UltraTool 上进行了多轮测试，涵盖了不同规模的采样数 ($m$) 和分支预算 ($b$)。

#### 6. 主要结论与发现
*   **熵是有效的诊断信号**：实验证明错误步骤的平均熵显著高于正确步骤，利用熵引导分支能有效定位错误。
*   **性能领先**：在 SLATE 上，EGB 的成功率比 ReAct 高出约 24.7%，比 LATS 高出 17.5%（在相同计算预算下）。
*   **计算效率高**：EGB 通过有针对性的分支，避免了 MCTS 那种穷举式的搜索，在保持高成功率的同时显著降低了推理延迟和 Token 成本。

#### 7. 优点
*   **评估严谨**：SLATE 填补了大规模工具使用中缺乏确定性、端到端评估的空白。
*   **搜索策略巧妙**：EGB 结合了不确定性量化，将搜索压力集中在“最可疑”的决策点上，非常适合长程任务。
*   **兼容性强**：既支持白盒模型的 Logits 分析，也支持黑盒模型的采样分析。

#### 8. 不足与局限
*   **合成数据偏差**：SLATE 是由 Claude 3.7 生成的合成数据，可能存在模型家族偏见。
*   **模拟器局限**：确定性模拟器无法完全模拟真实 API 的网络延迟、瞬时故障等非确定性行为。
*   **采样成本**：在黑盒模式下，为了估计熵需要进行多次采样（如 $m=10$），这会增加初始执行阶段的 API 调用成本。
*   **环境假设**：实验假设环境支持回滚/重试（Backtracking），在某些不可逆的真实生产环境中应用受限。

## 摘要
前沿大语言模型（LLMs）能够导航复杂的网站，但其高昂的成本以及对第三方 API 的依赖使得本地部署变得不切实际。我们引入了 Agent-as-Annotators，这是一个通过类比人类标注角色来结构化 Web 智能体合成轨迹生成的框架，利用模块化的 LLM 组件取代了任务设计者（Task Designer）、标注者（Annotator）和监督者（Supervisor）。我们使用 Gemini 3 Pro 作为教师模型，在六个 Web 环境中生成了 3,000 条轨迹，并在通过质量过滤的 2,322 条轨迹上，通过纯监督学习微调了一个 9B 参数的学生模型。所得模型在 WebArena 上达到了 41.5% 的成功率，在相同的评估协议下超过了 Claude 3.5 Sonnet (36.0%) 和 GPT-4o (31.5%) 等闭源模型，并且几乎是之前最佳开源权重结果（Go-Browse, 21.7%）的两倍。该模型的能力可以迁移到未见过的环境，在 WorkArena L1（一个训练期间从未见过的企业平台）上获得了 18.2 个百分点的提升，并在另外三个基准测试中取得了一致的改进。消融实验证实，流水线中的每个组件都做出了重要贡献，其中评判过滤（Judge filtering）、评估提示（evaluation hints）和推理轨迹（reasoning traces）各自都带来了可衡量的提升。这些结果表明，利用单个前沿教师模型进行结构化轨迹合成，足以产生具有竞争力的、可本地部署的 Web 智能体。项目页面：https://agent-as-annotators.github.io

## Abstract
Frontier LLMs can navigate complex websites, but their cost and reliance on third-party APIs make local deployment impractical. We introduce Agent-as-Annotators, a framework that structures synthetic trajectory generation for web agents by analogy to human annotation roles, replacing the Task Designer, Annotator, and Supervisor with modular LLM components. Using Gemini 3 Pro as teacher, we generate 3,000 trajectories across six web environments and fine-tune a 9B-parameter student with pure supervised learning on the 2,322 that pass quality filtering. The resulting model achieves 41.5% on WebArena, surpassing closed-source models such as Claude 3.5 Sonnet (36.0%) and GPT-4o (31.5%) under the same evaluation protocol, and nearly doubling the previous best open-weight result (Go-Browse, 21.7%). Capabilities transfer to unseen environments, with an 18.2 percentage point gain on WorkArena L1 (an enterprise platform never seen during training) and consistent improvements across three additional benchmarks. Ablations confirm that each pipeline component contributes meaningfully, with Judge filtering, evaluation hints, and reasoning traces each accounting for measurable gains. These results demonstrate that structured trajectory synthesis from a single frontier teacher is sufficient to produce competitive, locally deployable web agents. *   **研究背景**：前沿大模型（如 GPT-4o, Claude 3.5）在网页导航任务上表现卓越，但由于 API 成本高昂、隐私风险以及无法本地部署，限制了其广泛应用。
*   **核心问题**：小型开源模型（约 9B 参数）与前沿模型之间存在巨大的能力鸿沟（在 WebArena 基准上差距超过 22%）。
*   **研究动机**：探索如何通过“智能体能力蒸馏”，利用前沿模型作为教师生成高质量训练轨迹，使小型模型在保持本地部署优势的同时，达到甚至超越闭源模型的水平。

### 2. 方法论：AGENT-AS-ANNOTATORS 框架
该框架通过模拟人类标注网页任务的三个角色，将合成轨迹生成过程模块化：
*   **任务设计者（Task Designer）**：
    *   **人格生成器（Persona Generator）**：生成多样化的用户背景（如数据科学家、设计师），以诱导产生不同分布的任务。
    *   **任务生成器（Task Generator）**：在环境中探索并合成任务意图，同时生成基于环境状态的**评估提示（Evaluation Hints）**。
*   **标注者（Annotator/Agent）**：由教师模型扮演，仅根据任务意图在环境中执行操作，生成交互轨迹。
*   **监督者（Supervisor/Judge）**：利用“评估提示”对轨迹进行自动评判，仅保留成功的轨迹用于后续训练。
*   **关键技术细节**：
    *   **SFT 训练**：在 2,322 条过滤后的成功轨迹上进行监督微调（SFT）。
    *   **推理轨迹（Reasoning Traces）**：在训练数据中保留教师模型的思考过程（`<thought>` 标签），实验证明这能显著提升学生模型的逻辑性。

*   **对比方法**：
    *   **闭源模型**：GPT-4o, Claude 3.5 Sonnet, Gemini 3 Pro。
    *   **开源基座**：Qwen3.5 系列（2B 到 27B）。

### 6. 主要结论与发现
*   **性能突破**：微调后的 **A3-Qwen3.5-9B** 在 WebArena 上达到 **41.5%** 的成功率，超过了 GPT-4o (31.5%) 和 Claude 3.5 Sonnet (36.0%)。
*   **泛化能力强**：在从未见过的企业平台 WorkArena L1 上提升了 **18.2%**，证明模型学到的是通用的网页交互原语，而非简单的环境记忆。
*   **质量胜过数量**：仅需约 2,300 条高质量过滤轨迹，即可让 9B 模型超越参数量大得多的模型。
*   **教师思考预算**：反直觉的是，减少教师模型的思考预算（Reduced Thinking）反而能生成更简洁、更易于学生学习的高质量轨迹。

### 7. 论文优点
*   **框架结构化**：将复杂的合成数据流水线映射到成熟的人类标注角色，具有很强的可解释性和复用性。
*   **评估提示（Hints）创新**：引入 Hints 辅助 Judge 评判，解决了网页任务最终状态难以自动判定的痛点。
*   **开源贡献**：提供了高性能的 9B 模型权重和完整的合成数据集，对开源社区贡献巨大。

### 8. 不足与局限
*   **环境多样性限制**：虽然泛化性不错，但训练环境仍局限于 WebArena 的 6 个站点，未来需扩展到更广阔的互联网规模。
*   **评判器偏差**：Judge 模块本身基于 LLM，可能存在假阳性（False Positive）风险，且未经过大规模人工校验。
*   **训练范式单一**：目前仅采用 SFT，尚未结合强化学习（RL）进行进一步的性能压榨。

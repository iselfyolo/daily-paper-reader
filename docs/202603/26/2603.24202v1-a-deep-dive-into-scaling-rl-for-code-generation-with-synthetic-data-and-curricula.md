## 摘要
强化学习（RL）已成为在监督微调之外改进大语言模型的一种强大范式，然而，在大规模场景下持续获得性能提升仍是一个公开挑战，因为数据的多样性和结构（而非单纯的数量）已成为限制因素。我们通过引入一种可扩展的多轮合成数据生成流水线来解决这一问题。在该流水线中，教师模型根据上下文中的学生表现总结迭代地改进问题，从而在无需对教师模型进行任何微调的情况下，生成结构化的难度梯度。与单轮生成相比，这种多轮方法显著提高了有效合成问题的产量，并自然地产生了“垫脚石”（即同一核心任务的更简单和更困难的变体），从而支持基于课程的学习训练。我们系统地研究了在 Llama3.1-8B Instruct 和 Qwen3-8B Base 模型系列中，任务难度、课程调度和环境多样性在强化学习训练过程中是如何相互作用的，并在 Qwen2.5-32B 上进行了额外的扩展实验。我们的结果表明，合成增强一致地提升了域内代码性能，并在大多数情况下提升了域外数学性能。此外，我们还就课程设计和数据多样性如何共同塑造强化学习训练动态提供了经验性见解。

## Abstract
Reinforcement learning (RL) has emerged as a powerful paradigm for improving large language models beyond supervised fine-tuning, yet sustaining performance gains at scale remains an open challenge, as data diversity and structure, rather than volume alone, become the limiting factor. We address this by introducing a scalable multi-turn synthetic data generation pipeline in which a teacher model iteratively refines problems based on in-context student performance summaries, producing structured difficulty progressions without any teacher fine-tuning. Compared to single-turn generation, this multi-turn approach substantially improves the yield of valid synthetic problems and naturally produces stepping stones, i.e. easier and harder variants of the same core task, that support curriculum-based training. We systematically study how task difficulty, curriculum scheduling, and environment diversity interact during RL training across the Llama3.1-8B Instruct and Qwen3-8B Base model families, with additional scaling experiments on Qwen2.5-32B. Our results show that synthetic augmentation consistently improves in-domain code and in most cases out-of-domain math performance, and we provide empirical insights into how curriculum design and data diversity jointly shape RL training dynamics.

*   **多样性受限**：真实代码竞赛题目的模板和结构相对固定，限制了模型的泛化能力。
论文旨在探索如何通过**合成数据**和**结构化课程学习**（Curriculum Learning）来确保持续的性能增益。

#### 2. 方法论：核心思想与关键技术
论文提出了一套可扩展的**多轮合成数据生成流水线**，核心思想包括：
*   **教师-学生迭代架构**：使用一个强大的教师模型（如 GPT-OSS 120B）生成初始题目，由学生模型尝试解决。教师模型根据学生的成功/失败总结（通过 In-Context Learning）迭代修改题目，调整难度。
*   **多轮生成（Multi-turn Generation）**：相比单轮生成，多轮迭代能显著提高有效题目的产出率（约 4 倍），并自然产生“阶梯式”任务链（Stepping Stones），即针对同一核心逻辑的简单、中等、困难变体。
*   **多样化环境**：设计了四种 RL 环境：
    1.  **Induction（归纳）**：程序合成。
    2.  **Abduction（溯因）**：给定函数和输出，预测输入。
    3.  **Deduction（演绎）**：给定函数和输入，预测输出。
    4.  **Fuzzing（模糊测试）**：寻找能使测试失败的边界输入。
*   **RL 算法**：采用异步变体 **GRPO**（Group Relative Policy Optimization），移除了 KL 散度正则项和标准差归一化，以适应大规模训练。

#### 6. 主要结论与发现
*   **合成增强有效性**：在真实数据基础上加入合成数据，能显著提升模型在 LCB 上的表现，甚至优于单纯增加三倍真实数据。
*   **难度“金发姑娘”原则**：**中等难度**的题目是 RL 训练的“甜点区”，能提供最佳的收敛速度与泛化平衡；简单题易导致过拟合，难题则因奖励稀疏导致训练不稳定。
*   **逆向课程的优势**：从较难或中等题目开始训练（逆向课程）可以防止模型过早对简单任务产生路径依赖，保持探索能力。
*   **环境多样性是新的扩展轴**：将数据预算分配到多个不同类型的环境（如溯因、演绎、模糊测试）比堆积单一环境的数据更能提升模型的泛化能力（尤其是数学能力）。

#### 7. 优点与亮点
*   **多轮生成机制**：通过教师反馈循环解决了合成数据质量低和难度失控的问题。
*   **系统性研究**：对 RL 训练中的难度、课程、多样性进行了多维度的解构，提供了清晰的工程指南。
*   **跨领域验证**：不仅关注代码，还验证了代码 RL 对数学推理能力的迁移效果。

#### 8. 不足与局限
*   **教师模型依赖**：该流水线高度依赖于一个极强的教师模型（120B 级别）来进行 In-Context 难度调整，对于缺乏超大模型访问权限的研究者较难复现。
*   **计算成本**：虽然比微调教师模型省力，但多轮生成和大规模 RL 训练依然消耗巨大的推理和训练算力。
*   **任务类型局限**：目前主要集中在算法竞赛类代码，对于系统级编程或长文本工程代码的适用性尚未验证。
